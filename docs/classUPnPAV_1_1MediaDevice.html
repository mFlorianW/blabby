This interface defines functions that every upnp media device should support.

modeWhich seek mode shall be used.
targetThe target of seek. The value depends on the seek modes.
Returns

Note
The following seek modes need the following target values. RelCount, => need an uint32 as target AbsCount, => need an uint32 as target RelCount, => need an uint32 as target TrackNr, => need an uint32 as target Frame, => need an uint32 as target TapeIndex, => need an uint32 as target AbsTime, => Timestamp in the format h.m.s.z RelTime, => Timestamp in the format h.m.s.z ChannelFreq, => float

◆ state()

MediaDevice::State UPnPAV::MediaDevice::state ( ) const
virtualnoexcept

Gives the current state of the device. The device will be updated with the lastChange event updates of the AVTransport service. If the device has no AVTransport service e.g. MediaServer the state will always be UPnPAV::MediaDevice::State::Idle.

◆ stateChanged

void UPnPAV::MediaDevice::stateChanged ( )
signal

This signal is emitted when the UPnPAV::MediaDevice changes it state. E.g. from Stopped to playing State changes only occur when the device has an AVTransport service. Devices without AVTransport service are going to stay in the MediaDevice::State::Stopped.
