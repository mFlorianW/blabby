void Multimedia::Renderer::initialize ( )
noexcept

Initializes the Mulitmedia::Renderer. During the initialization the fetches all informations like, supported protocols for working properly. Without calling the initialization the Multimedia::Renderer does nothingl. The signal Mulitmedia::Renderer::initializationFinished is emitted on success. The signal Mulitmedia::Renderer::initializationFailed is emitted with a the error Information.

◆ resume()

void Multimedia::Renderer::resume ( )
noexcept

Resumes the playback when the Multimedia::Renderer has the state Mulitmedia::Renderer::State::Stopped or Mulitmedia::Renderer::State::Paused. If the device is in the stopped state the playback starts all over again and for the pause state the track resumes at the pause state. In all other cases the function does nothing.

◆ stop()

void Multimedia::Renderer::stop ( )
noexcept

Stops or pause the current playback. Pause is the preferred way to stop the playback but not every device support this feature. So the pause is called when the device supports it in all other cases stop is called. If the playback is not active nothing happens. On success the signal Multimedia::Renderer::stateChanged is emitted. The new state then should be be stopped or paused.
